Implicit Conversion(Boooo Hoooooo) JavaScript implicitly converts the type of a object in many scenarios. One of the most common scenarios it while coercion. When using the '==' operator JavaScript implicitly coerce's the type of the two operands. This totally freaks many of the controlling freaks out there as they cannot control what's happening. Many even suggest not to use '==' operator. But that imho is sad because if you are a programmer you should understand what exactly is happening and use this automatic conversion to your use. JavaScript has basically two types of values. Primitive: Null: The valuenull. Undefined: The valueundefined. Number: All numbers, such as0 and 3.14. AlsoNaN, andInfinity. Boolean: The valuestrue and false. String: All strings, such as"foo" and "". Non-Primitive: All other values are non-primitive, including arrays, functions, and plain old objects. So what happens when using '==' operator for comparison between different type is that either one or both operands will be converted to a single type and they will compared. This is a very confusing thing if you don't know what exact type and value the operands will be converted to.
