So in this long journey the basic criteria is that you have a passion to learn. So where better to start learning than HTML. HTML(Hyper Text Markup Language) is not a programming language. Rather it is a language used to structure websites. There are something called tags which are used to structure the website. Example- <html>
<body>

<h1>My new page.</h1>

<p>I can write something here which people will want to read.</p>

</body>
</html>
The tags, such as <html>,<body> are defined within angular brackets.The scope of the tag is from the  the end tag defined by using a '/' before the name of the tag. So as seen in the example the first tag is <html> tag. This is used to designate all the stuff written till </html> is specified using HTML format. So basically the tags are like a fence on a ranch. So if you forget to close your fence you better be ready to be stamped upon by a lot of code. Mooooo!!!!! Tip - Always check if you have closed all the tags. The tags can have attributes which will describe that specific tag. The attributes are specified within quotes. Example-
<p id='my_para_id' name='my_para_name'>........</p> There are a lot of tags to be learnt on HTML. One good resource to refer for the semantics of HTML is http://www.w3schools.com. But designing a website is not only about structuring. You need to style it i.e make it presentable. That's where CSS comes into picture. HTML is blueprint. You can visualize and structure a house using the blueprint. But to live in the house you would want it to be painted, decorated and with all the rooms in the right place. So CSS is fundamentally like wood/wallpaper/furniture. You can position,color and do lots more with CSS.
Basic Css Syntax[From w3schools.com]
The image above shows the basic CSS syntax. There are 3 ways of placing CSS,
  • External Style sheet : This is used when a lot many web pages are to be styled similarly. Here are the CSS styles are specified in a single file, a .css file, which will be referred by all the pages which are to look similar. The benefit of using this (I am sure the programmers out there are comparing this to a common function) is that you need to repeat the same styles over and over again. Also if the look of the web pages are to be changed, then changing in one place(i.e the css file) will replicate the changes across all the pages.
  • Internal Style sheet : This is used when one particular web page is to be styled differently from the others. Here the the styles are specified in a specific section of the HTML. The styles specified here are local to the HTML document in which the styles are specified.
  • Inline : This is used when only when one element(tag) is to be styled differently.
There is no hard and fast rule for placing the CSS. But normally External Style Sheet is the best way to go. Also the performance is improved in case of external style sheet as the browser can cache the style and need to reload the whole file every time. But ultimately I guess it just depends on what the situation requires to take a call on where the style sheet should be specified. Tip - Using external style sheet also helps you to inherit style from parent elements which means even lesser lines of code. Being lazy is the way to being efficient. Also it's a very good practice to use comments to describe the HTML and CSS you have written. This makes your code all the more readable to you and also to others.(For some reason people just hate to comment their code/markup. Designing good web pages is like art. You need to love what you are doing to succeed in it. You also need to be selfless(Na don't worry. I ain't asking you to sell your Ferrari and become a monk.) Selfless as in stop thinking what you want out of you website and start thinking of what the people who will be using it want. There a lot of websites out there which have crammed up so much of information on their sites that looking at them makes you remember the last time you overeaten the thanks giving turkey. You eyes feel blinded not from the brilliance of the site but from the sheer amount crazy glitter and you 'll be scrambling frantically to get the hell out that web page. The simple philosophy I will follow is simple is beautiful. May be not HOT but definitely beautiful. Some tips which I found out are,
  • Keep is simple and straight forward. Do not display more than required.
  • Represent all the stuff uniformly so that the users don't get confused.
  • Ensure proper color contrast and text readability.
  • Ensure a proper site map.
  • Keep working and iterating on you web page.
  • Get criticisms from people and take in all that on which you can improve your awesome website.
